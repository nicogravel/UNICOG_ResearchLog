Create a Python package

Create a setup.py file in the root of your project:

cd UNICOG_ResearchLog
touch setup.py

and add the following to setup.py:

from setuptools import find_packages, setup

setup(
    name='myCodeIsYourCode',
    packages=find_packages(),
)

Create myCodeIsYourCode directory an add empty init.py file to it, together with a python file that prints â€œhello worldâ€ to your package:

mkdir myCodeIsYourCode
cd myCodeIsYourCode
touch __init__.py
echo "print('hello world')" > helloworld.py

Go to the root directory and install your package from the root directory:

cd ..
pip install -e .
sphinx-apidoc -f -o docs_local/source myCodeIsYourCode

Try it:

python

>>> import myCodeIsYourCode.helloworld
hello world
>>> exit()


Generate project and code documentation using Sphinx

The folder docs will be used to generate sphinx docs, cd into this directory and run:

pip install sphinx
pip install sphinxcontrib-bibtex
pip install sphinxcontrib-napoleon
pip install -U sphinxcontrib-matlabdomain
pip install sphinx-autodoc-typehints

sphinx-quickstart
make html

We can then install sphinx.ext.napoleon, a sphinx plugin, to automatically read the doctsrings we have written as â€˜readmeâ€™sâ€™ in the top of our python scripts, and add that content to the package dictionary of functions. To do so:

Edit myCodeIsYourCode.rst: add :noindex: to the end of the file, as follows:

   :members:
   :undoc-members:
   :show-inheritance:
   :noindex:

Now enjoy building up your python package!
